## 2026-01-23: CE-003 - Undo/Redo Functionality

  - historyRef: stores canvas JSON snapshots (max 50 states)
  - historyIndexRef: tracks current position in history
  - isRestoringRef: prevents saving state during undo/redo
  - canUndo/canRedo state for button enablement
- Implemented saveHistoryState function:
  - Serializes canvas to JSON
  - Removes future states when new action taken

**Status:** COMPLETE

**Changes:**
- Created canvas-server.ts with server functions:

  - `saveCanvasState` - server function wrapper
  - `getCanvasStateInternal()` - fetch canvas JSON from DB
  - `getCanvasState` - server function wrapper
  - `CanvasData` type for JSONB storage
  - `CanvasStateRecord` interface for typed returns
- Updated FabricCanvas component with persistence props:
  - `initialCanvasData` - restores saved state on mount via loadFromJSON
  - `onCanvasChange` - callback for debounced auto-save
  - `SaveStatus` type exported for parent components
- Added save indicator UI to toolbar:
  - "Saving..." with spinner during save
  - "Saved" with checkmark on success (auto-hides after 2s)
  - "Error" with alert icon on failure (auto-hides after 5s)
- Implemented debounce utility (1-second delay)
- History is reset after restoring initial data

**Files changed:**
